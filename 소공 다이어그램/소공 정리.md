
✅ 1. 시퀀스 다이어그램은 “유스케이스별 대표 흐름”만 그리면 됨
즉, 전체 시스템 다 그리는 게 아니라
각 유스케이스에서 하나의 대표 성공 시나리오만 선택해서 그리면 충분해요.
예시로 아래 6개 중 3~4개만 선택하면 합격/만점 수준:
추천 우선순위	유스케이스
⭐⭐⭐	서비스 시작 (클라우드 연결 + 템플릿 추천)
⭐⭐⭐	자연어 기반 파일 검색
⭐⭐⭐	기존 파일 분류 실행 (템플릿 기반 정리)
⭐⭐	신규 파일 자동 저장
⭐	파일 제거 및 최적화
⭐	템플릿 생성/공유/복제

구독/결제는 이전에 이미 했으니 패스해도 OK.

✅ 2. 시퀀스 다이어그램 만들 때 따라야 할 공식
1) 등장 객체(라이프라인) 식별
유스케이스에서 등장한 역할들을 객체로 변환
예시: “기존 파일 분류”
User
UI
FileScanner
MetadataService
RuleEngine
CloudStorageAPI
LogService

2) 메인 시나리오 메시지 순서대로 배치
예:
User → UI : 파일 정리 시작
UI → FileScanner : 파일 목록 요청
FileScanner → CloudStorageAPI : 파일 리스트 가져오기
CloudStorageAPI → FileScanner : 파일 목록 반환
FileScanner → MetadataService : 메타데이터 추출
MetadataService → RuleEngine : 템플릿 규칙 평가
RuleEngine → CloudStorageAPI : 이동/정리 명령
CloudStorageAPI → LogService : 작업 결과 기록
UI → User : 정리 결과 보여주기

3) alt / opt / loop 프레임
필요한 경우만 넣기
상황	UML 키워드
검색 결과 없음	alt
파일 너무 많음	loop
권한 부족	alt
모든 예외 넣을 필요 X → 한두 개만 넣어도 충분.

✅ 3. 이제 상태 다이어그램은 무엇을 대상으로?
상태 다이어그램은 객체의 상태 변화를 보여주는 것이므로
시스템 전체가 아니라 핵심 엔티티만 하면 됩니다.
추천 상태 다이어그램 대상
대상	이유
파일(File)	업로드 → 분석 → 분류 → 보류/복구
템플릿(Template)	생성 → 공유 → 복제 → 수정
구독(Subscription)	Inactive → Active → PendingCancel → Cancelled

2개만 해도 충분
3개 하면 완벽
