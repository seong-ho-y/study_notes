# 📌 **2. Unreal Editor는 반드시 종료한 상태에서 pull 하세요.**

에디터가 켜져 있으면 맵/에셋 파일을 잠가서  
pull 중 파일 충돌이 발생할 수 있습니다.

---

# 📌 **3. 절대로 맵 파일이나 에셋을 직접 삭제하지 마세요.**

Git이 파일을 직접 수정했다고 인식해서  
pull 받더라도 정상적으로 업데이트되지 않을 수 있습니다.

---

# 📌 **4. 자신의 브랜치를 최신 상태로 업데이트하는 절차**

`git switch <자기브랜치>
git pull origin NewMain`

예시:

`git switch EnemyAI git pull origin NewMain`

이 두 줄이면, NewMain에 적용된 모든 변경사항이  
여러분의 브랜치에 자동으로 반영됩니다.

---

# 📌 **5. 변경 충돌(Conflict)이 뜰 경우 절대 임의 해결하지 말고 바로 알려주세요.**

특히 .umap / .uasset 충돌은 수동으로 해결 불가능합니다.  
이 경우 담당자(예: 유성호)에게 바로 알려주세요.

---

# 📌 **6. Git LFS 필수 설치 확인 (처음 설정하는 사람만)**

`git lfs install
git lfs pull`

이 과정을 하지 않으면  
에셋이 비어 보이거나 맵이 안 열리는 문제가 발생합니다.

---

# 📌 **7. Pull 후 Unreal Editor 실행**

위 동작을 모두 완료한 뒤 에디터 실행하면  
모든 맵/에셋이 정상 로딩됩니다.
